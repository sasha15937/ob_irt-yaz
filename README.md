# Создание многосекционного сайта

_За это задание можно получить до 6 баллов, влиять на количество будет качество кода, наполнение и дизайн_

## Цель

Сверстать многосекционный сайт на свободную тему, применив семантическую вёрстку, BEM-нейминг и rem hack с адаптивностью на двух брейкпоинтах.

---

## Тема сайта

Выберите любую тему, которая вам интересна. Если идей нет — вот несколько вариантов:

- Музыкальная школа или студия
- Кофейня или ресторан
- Фитнес-клуб
- Фотограф или дизайнер (портфолио)
- Магазин растений или книг
- Путешествия и туризм
- Игровая студия или киберспорт

---

## Структура сайта

Ниже описаны секции, которые должны быть на сайте. Некоторые из них обязательны, остальные — на ваше усмотрение. Вы сами решаете, что писать внутри каждой секции, какие тексты использовать и как всё выглядит визуально.

### Header — обязательно

Шапка сайта с навигацией. Должна быть зафиксирована при скролле (см. теорию ниже).

Пример содержимого: логотип или название, ссылки на секции страницы, кнопка действия.

---

### Hero — обязательно

Главный экран, первое что видит пользователь. Должен занимать значительную часть экрана и сразу передавать суть сайта.

Пример содержимого: заголовок, подзаголовок, кнопка.

---

### Секция с карточками

Сетка карточек. Минимум три карточки. Что внутри каждой карточки — решаете сами.

Пример для кофейни: карточки меню с названием напитка, описанием и ценой.
Пример для портфолио: карточки проектов с названием и описанием.
Пример для фитнес-клуба: карточки тарифов.

---

### Галерея — обязательно

Сетка изображений. Минимум шесть изображений.

---

### Дополнительные секции

Добавьте одну или несколько секций на своё усмотрение, если они подходят по теме. Например:

- Секция «О нас» или «О себе»
- Отзывы
- Команда
- Преимущества
- Контакты или форма обратной связи (без JS, просто вёрстка)
- FAQ

---

### Footer — обязательно

Подвал сайта. Пример содержимого: копирайт, ссылки, контакты, социальные сети.

---

## Технические требования

- Семантическая вёрстка: использовать подходящие теги — `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<article>`, `<ul>` и другие по смыслу
- BEM: все классы именуются по методологии BEM
- Rem hack: базовый `font-size` на `html` выставлен через rem hack, все размеры в проекте указаны в `rem`. Должно отрабатывать минимум на двух брейкпоинтах

---

## Теория — фиксированный хэдер

Чтобы шапка оставалась видна при скролле, используют одно из двух CSS-свойств: `fixed` или `sticky`. Они похожи, но работают по-разному.

### position: fixed

Элемент вырывается из потока документа и позиционируется относительно окна браузера. Он всегда остаётся на одном месте, даже если страница скроллится.

```css
.header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 100;
}
```

Важный момент: так как элемент вырван из потока, следующий за ним контент «заедет» под шапку. Чтобы это исправить, добавьте отступ сверху к первой секции после хэдера:

```css
.hero {
  padding-top: 80px; /* высота вашего хэдера */
}
```

### position: sticky

Элемент остаётся в потоке документа и ведёт себя как обычный элемент — до тех пор, пока при скролле не достигнет указанной позиции. После этого он «прилипает» и остаётся на месте.

```css
.header {
  position: sticky;
  top: 0;
  z-index: 100;
}
```

В отличие от `fixed`, здесь не нужно компенсировать отступ — шапка не вырвана из потока.

### Что выбрать

Оба варианта решают задачу. `sticky` проще в использовании, потому что не требует компенсации отступов. `fixed` даёт больше контроля, если нужно точно управлять позиционированием.

## Как сдавать

- Создайте форк репозитория в вашей организации с названием `название-репозитория-вашафамилия`
- Используя ветку `wip` сделайте задание
- Зафиксируйте изменения в вашем репозитории
- Когда работа будет готова — создайте пул-реквест из ветки `wip` в ветку `main` и укажите меня (`ktkv419`) как reviewer

Не мержите пул-реквест самостоятельно — это сделаю я после проверки.
